Introduction to Xcode and SwiftUI

The video for this material is available on Canvas. In the video, I discuss Xcode and the basics of SwiftUI. Here are a couple of the main ideas.

  • The user interface of an iOS application is made up of views, which are objects of types that conform to the View protocol.

  • Conforming to the View protocol requires implementing a single computed property body of type some View, which is a special Swift way of saying "any type which is a view."

  • Building views in in Swift UI is an example of declarative programming. The main difference between this and what we have seen is that there is no notion of time. The views describe the state of the UI, independent of time, but dependent on some other structure (class actually, more on that later) that keeps track of the functionality of the code.

  • There are a number of basic views that we will use:

    • Text(_:), a view containing the input text
    • Rectangle(), a view in the shape of a rectangle
    • RoundedRectangle(cornerRadius:), a view in the shape of a rectangle with rounded edges.
    • Circle(), a view in the shape of a circle
    • Spacer(), a view with empty space

    Different views have different behaviors with respect to how they fill the space on the screen. Text views try to take as little space as possible, whereas Rectangles() try to take as much space as possible. We will discuss this a bit more when we talk about layouts.

  • Views can be combined using container views. You can create custom container views, but we will mostly be using some standard built in ones.

    • ZStack { ... }, a view containing several views layers on top of each other
    • HStack { ... }, a view containing several views side by side, from left to right
    • VStack { ... }, a view containing several views arranged from top to bottom

    The views appearing in the brackets each appear on their own line. The brackets are an example of a [trailing closure], which we have not formally discussed but you should read about.

  • Views have methods for describing their features (note that we don't say they change their features, since this would imply a notion of time).

    • .foregroundColor(:_) describes the color of a view. It takes a Color, for which there are static variables for standard colors like .red and .blue.
    • .font() describes the font used within the view. It takes a Font, for which there are static variables for standard system fonts like .largeTitle
    • .frame(...) describes the size and shape of the view.
    • .onTapGesture { ... } describes what should happen in the application if the view is pressed by the user.
