Lab 4: Ordered Dictionaries

This lab is an exercise in structure design and data abstraction. You will fill in a partial definition of an ordered dictionary structure, a data type with key-value pairs ordered by when the keys were added. So pairs for newly added keys appear at the end of the order but updating values at a given key does not affect the order.

There are two of data representations in the structure, one is just an array if key-value pairs called data, the other has two parts, a dictionary, and an array of keys, data1 and data2. For each of these data representations you will define the getter/setter functions given in the structure.

Everything else you have to fill in is on the other side of the abstraction barrier and should not depend on the data representation, only on the getter/setter functions. You should be able to switch out the representation and the getter/setter functions without changing anything else and still pass all the tests. These methods either already appear in the Dictionary type or (I believe) are specified clearly enough by their name. If this is not the case, please let me know.

Note. You might have noticed that the variable S for the OrderedDictionary type is annotated by : Hashable. This is an example of a type constraint, the topic for the next meeting. You should be able to ignore it for the sake of this lab. It basically means that we can use S as keys in dictionaries, like data1.

Subscripts

There is one Swift tool we haven't covered that we will be using for this lab. In Swift, the syntax for indexing an array or a dictionary (e.g., l[2] or d["key"]) is not just specialized syntax for these objects. This syntax can be used for any type via subscripts. To do this, we just have to include a subscript method in the type.

subscript(i1: Type1, ..., iK: TypeK) -> TypeO {
    get {
        // Code that is run when evaluating an expression like x[v1, ..., vk]
    }
    set(newValue) {
       // Code that is run on an assignment like x[v1,...,vk] = newValue
    }
}

Subscripts behave a lot like computed properties. The get methods is called when the subscript is used in an expression and the set function is used when the subscript is used in the subject of an assignment.

We will use subscripts so that we can index our ordered dictionaries like the built-in dictionaries.
